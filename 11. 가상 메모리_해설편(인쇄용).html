세트 1 · 정답과 해설

1번 — 정답 ③ 2문단에 "페이지 번호는 프로그램이 주기억 장치의 어느 자리에 올라가더라도 변하지 않는다"라고 못 박혀 있어요. 그런데 ③은 "어느 자리에 올라가는가에 따라 달라진다"라며 정반대로 말했으니 적절하지 않아 정답입니다. ①은 1문단, ②는 2문단, ④는 3문단, ⑤는 4문단 내용과 그대로 맞아요.

2번 — 정답 ② ㉠은 "페이지 번호와 페이지 프레임 번호의 대응을 따로 기록해 두는 표"로, 컴퓨터가 "실제 자리를 찾아"가는 데 쓰입니다(3문단). 이를 정확히 옮긴 ②가 정답이에요. ①은 페이지의 크기·개수가 "달라지지 않는다"(2문단)는 내용과 어긋나고, ③은 "연속한 프레임에 올라가는 것은 아니"라는 진술과 충돌합니다. ④는 표가 '올라가지 않은 페이지'를 모은다는 근거가 지문에 없고, ⑤는 한 프레임에 한 페이지만 올라간다는 4문단과 정면으로 어긋나요.

3번 — 정답 ④ 프로세스(㉡)에 대한 추론입니다. 4문단 끝에 "당장 필요한 페이지의 수가 빈 프레임의 수를 넘지 않는 한 프로그램은 정상적으로 실행된다"라고 했죠. 이를 뒤집어 말한 ④가 정답입니다. ①은 "반드시 연속한 페이지 프레임에 올라가는 것은 아니"라는 3문단과 충돌하고, ②는 '필요한 페이지 수'가 아니라 '전체 페이지 수'로 잘못 따졌어요(전체가 많아도 실행될 수 있죠). ③은 "같은 프레임을 동시에 차지하는 일은 없다"와 반대, ⑤는 "페이지의 개수는 달라지지 않는다"(2문단)와 어긋납니다.

4번 [3점] — 정답 ⑤ 지문은 페이지가 "비어 있는 프레임이면 어디든 올라갈 수 있다"라고 했지, '페이지 번호와 같은 번호의 프레임에 올라가야 한다'고 한 적이 없어요. 〈보기〉에서도 을의 0번 페이지가 3번 프레임에 올라갔으니, 번호 일치 규칙은 성립하지 않습니다. 그래서 ⑤가 적절하지 않아 정답이에요. ①②는 '빈 프레임 자유 적재'·'비연속 적재'를, ③은 대응표의 역할을, ④는 7개 프레임 중 5개가 차서 1·6번 두 자리가 남는다는 계산을 정확히 적용한 적절한 이해입니다.

❀ 개념 깊이 공부하기
핵심 용어
  • 페이징: 프로그램을 같은 크기의 페이지로 나눈 뒤, 필요한 페이지만 주기억 장치에 올리는 가상 메모리 실현 방법.
  • 페이지: 프로그램을 같은 크기로 자른 작동 단위. 0부터 번호가 붙어요.
  • 페이지 프레임: 주기억 장치에서 페이지가 들어갈 자리. 역시 0부터 번호가 매겨집니다.
  • 대응표: 페이지 번호와 페이지 프레임 번호가 어떻게 짝지어졌는지 적어 둔 표. 이게 있어야 실제 자리를 찾아가요.
개념 한눈에
[분할 단계]  프로그램 ──같은 크기로 자름──▶  페이지 0,1,2,3,4 …
                                              (개수·크기 고정, 번호 불변)
        │
        ▼
[적재 단계]  당장 필요한 페이지만 골라  ▶  주기억 장치의 빈 페이지 프레임에 올림
                                              (빈 자리면 어디든·비연속 가능)
        │
        ▼
[대응표]   페이지 번호 ↔ 프레임 번호 를 기록  ▶  컴퓨터가 실제 자리를 찾아감
확인 질문
페이지 0번과 1번은 꼭 프레임 0번과 1번에 나란히 올라가나요?
아니요. 비어 있는 프레임이면 어디든 올라갈 수 있고, 번호가 연속한 페이지가 연속한 프레임에 놓이지 않아도 됩니다.
페이지가 프레임보다 많으면 프로그램이 못 돌아가나요?
아니요. 전체 페이지가 많아도, 당장 필요한 페이지 수가 빈 프레임 수 이하이면 정상적으로 실행됩니다.
어떤 페이지가 어느 프레임에 있는지 컴퓨터는 어떻게 아나요?
페이지 번호와 프레임 번호의 대응을 적어 둔 표를 참고해서 찾아갑니다.

세트 2 · 정답과 해설

5번 — 정답 ④ 3문단에 가상 메모리는 "별도의 새 장치가 아니라, 보조 기억 장치와 주기억 장치를 한데 묶어 운영하는 방식"이라고 분명히 적혀 있어요. 그런데 ④는 "별개로 새로 추가되는 기억 장치"라고 했으니 정반대라서, 일치하지 않아 정답입니다. ①은 1문단, ②는 2문단, ③은 1문단, ⑤는 [A]와 그대로 맞아요.

6번 — 정답 ④ [A]의 마지막 문장에 "주기억 장치의 용량보다 큰 프로그램도 필요한 부분만 나누어 올려 가며 실행할 수 있다"라고 했죠. 이를 옮긴 ④가 정답입니다. ①은 페이지가 "같은 크기"라는 점과 어긋나고, ②는 필요하지 않은 페이지는 "보조 기억 장치에 그대로 두"는 것이지 주기억 장치에 미리 올려 두지 않습니다. ③은 페이지를 "페이지 프레임으로 불러서 사용한다"와 충돌하고, ⑤는 "별도의 대응 정보로 관리된다"는 [A] 진술과 어긋나요.

7번 [3점] — 정답 ⑤ 가상 메모리의 핵심은 "당장 필요한 부분만 주기억 장치에 올린다"는 것이에요. 그래서 두 프로그램의 '모든 페이지'가 동시에 주기억 장치에 올라가 있어야 한다는 ⑤는 가상 메모리의 원리를 정면으로 어겼으니 적절하지 않아 정답입니다(전체 용량이 주기억 장치보다 크니 애초에 모두 올릴 수도 없죠). ①은 '일부만 적재', ②는 '나머지는 보조 기억 장치에 보관', ③은 '페이지 프레임에서 처리', ④는 '필요에 따라 적재 구성이 바뀜'을 정확히 적용한 적절한 반응입니다.

8번 — 정답 ② ⓑ '사용하다'는 '어떤 목적으로 쓰다'라는 뜻인데, '부리다'는 사람이나 동물을 마음대로 시켜 일하게 한다는 뜻이라 '보조 기억 장치를 메모리처럼 ~한다'라는 맥락에 호응하지 않아요. 그래서 바꿔 쓰기에 적절하지 않은 ②가 정답입니다. ① 수행하는→해내는, ③ 요구해도→원해도, ④ 이용한다→살린다, ⑤ 나누어→갈라는 모두 문맥에서 자연스럽게 호응합니다.

❀ 개념 깊이 공부하기
핵심 용어
  • 보조 기억 장치: 용량이 크고 프로그램·데이터를 영구히 보관하는 장치. 가상 메모리에서 '책장' 역할이에요.
  • 주기억 장치: 용량은 작지만 당장 처리할 내용을 담는 장치. '메모리'라 불리는 '책상'이에요.
  • 프로세스: 주기억 장치에 올라와 실행 중인 프로그램.
  • 가상 메모리: 새 장치가 아니라, 두 기억 장치를 묶어 보조 기억 장치 일부를 메모리처럼 쓰는 '운영 방식'.
개념 한눈에
구성 요소 용량 맡은 역할
보조 기억 장치 큼 프로그램 전체를 영구 보관
주기억 장치 작음 당장 쓰는 페이지를 담아 처리
가상 메모리(운영 방식) — 두 장치를 페이징으로 맞물려, 작은 메모리로 큰 프로그램 실행

헷갈리기 쉬운 점: 가상 메모리는 '장치'가 아니라 '방식'이에요. 보조·주기억 장치를 새로 사 붙이는 게 아니라, 이미 있는 두 장치를 똑똑하게 묶어 쓰는 운영 기법이라는 점을 꼭 기억하세요.

확인 질문
가상 메모리는 컴퓨터에 새로 추가하는 기억 장치인가요?
아니요. 보조 기억 장치와 주기억 장치를 한데 묶어 운영하는 방식입니다.
보조 기억 장치와 주기억 장치는 각각 무슨 일을 하나요?
보조 기억 장치는 프로그램 전체를 영구 보관하고, 주기억 장치는 당장 쓰는 페이지를 담아 처리합니다.
주기억 장치보다 용량이 큰 프로그램도 실행할 수 있나요?
네. 필요한 페이지만 나누어 올려 가며 실행하면 되기 때문입니다.

세트 3 · 정답과 해설

9번 — 정답 ④ 1문단에 페이지 폴트는 "모든 페이지가 주기억 장치에 올라가 있는 것은 아니므로" 일어난다고 했어요. 그런데 ④는 "모든 페이지가 주기억 장치에 올라가 있기 때문"이라며 정반대로 말했으니 적절하지 않아 정답입니다. ①은 1문단 정의, ②는 1문단, ③은 2문단(빈 프레임이 있으면 교체 없이 적재), ⑤는 2문단(대응표 갱신)과 맞아요.

10번 [3점] — 정답 ③ ⓒ는 ⓑ('빈 페이지 프레임이 있는가?')에서 '예'로 간 경우, 곧 빈 프레임이 있을 때 "그 프레임에 페이지를 불러와 올린다"는 동작이에요(2문단과 일치). 그래서 ③이 정답입니다. ①은 ⓐ '아니요'가 '올라와 있지 않은' 경우이므로 반대, ②는 ⓑ에 도달했다는 것 자체가 ⓐ에서 '아니요'(=페이지 폴트)로 왔다는 뜻이라 틀렸어요. ④는 ⓓ가 빈 프레임이 '없을' 때의 동작이므로 반대, ⑤의 ⓔ는 이미 올라와 있는 페이지를 처리하는 경우라 새로 불러오는 일이 없습니다.

11번 — 정답 ① 3문단에 "한 페이지를 내보내 자리를 비운 다음, 그 자리에 필요한 페이지를 불러온다"라고 했죠. 이 순서를 옮긴 ①이 정답입니다. ②는 내보낸 페이지가 "보조 기억 장치에 그대로 남아" 다시 돌아온다는 진술과 충돌하고, ③은 "한동안 쓰이지 않은 페이지를 먼저 내보내는 기준이 흔히 쓰인다"와 반대예요. ④는 교체가 잦으면 이동이 "늘어"난다는 4문단과, ⑤는 곧 쓰일 페이지를 내보내면 "금세 다시 불러와야" 한다는 진술과 정면으로 어긋납니다.

12번 — 정답 ② 〈보기〉의 문제는 '곧 다시 쓰일 페이지'가 자꾸 내보내져 같은 페이지가 들고 나기를 거듭하는 것이에요. 지문은 "교체 기준을 잘 정하여 곧 쓰일 페이지는 되도록 주기억 장치에 남겨 두는 것이 중요하다"라고 해법을 제시했죠. 이를 옮긴 ②가 정답입니다. ①은 한 프레임에 한 페이지만 올라간다는 원리에 어긋나고, ③은 필요한 페이지를 안 불러오면 처리 자체가 안 되며, ④의 대응표 갱신은 지연의 원인이 아니에요. ⑤는 불필요한 교체를 늘려 오히려 더 느려지게 합니다.

13번 — 정답 ⑤ ㉤ '더해지다'는 '어떤 것에 무엇이 보태어지다'라는 뜻으로, '첨가(添加)되다'와 자연스럽게 호응해 바꿔 쓰기에 적절하므로 정답입니다. ① ㉠ '없는'은 '있지 않은' 상태이므로 '부재한'으로 바꾸면 어색하고('부재'는 주로 사람·자리에 씀), ② '찾아내다'는 숨은 잘못을 들춘다는 뜻의 '색출'과 맥락이 어긋나며, ③ ㉢ '불러오는'을 '발생하는'으로 바꾸면 뜻이 달라지고, ④ ㉣ '찾아가다'는 시간이 닥쳐온다는 '도래'와 호응하지 않습니다.

  • 페이지 폴트: 중앙 처리 장치가 필요로 하는 페이지가 주기억 장치에 없는 상태.
  • 페이지 교체: 빈 프레임이 없을 때, 한 페이지를 내보내 자리를 비우고 다른 페이지를 들이는 일.
  • 교체 기준: 어느 페이지를 내보낼지 정하는 기준. 한동안 안 쓰인 페이지를 먼저 내보내는 방식이 흔해요.
  • 대응표 갱신: 페이지를 새로 올린 뒤, 페이지와 프레임의 짝을 새로 기록하는 일.

확인 질문
페이지 폴트는 왜 생기나요?
모든 페이지가 주기억 장치에 올라가 있지는 않아서, 필요한 페이지가 주기억 장치에 없을 때 생깁니다.
빈 페이지 프레임이 없으면 어떻게 하나요?
이미 올라간 페이지 하나를 보조 기억 장치로 내보내 자리를 비운 뒤, 그 자리에 필요한 페이지를 불러옵니다(페이지 교체).
내보낸 페이지는 사라지나요?
아니요. 보조 기억 장치에 그대로 남아, 다시 필요해지면 같은 과정을 거쳐 주기억 장치로 돌아옵니다.
