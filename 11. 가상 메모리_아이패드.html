길잡이 ―
이 개념, 쉽게 알아보기

여러분, 책상이 좁아서 책을 다 펼쳐 놓지 못해 본 적 있나요? 시험공부를 하려는데 국어·수학·영어 책에 문제집까지 한꺼번에 펴 두려면 책상이 모자라죠. 그럴 때 우리는 어떻게 하나요? 당장 푸는 책 한두 권만 책상에 올려 두고, 나머지는 책장에 꽂아 둡니다. 그러다 다른 과목이 필요해지면 책장에서 꺼내 오고, 다 본 책은 도로 꽂아 두고요. 컴퓨터도 똑같은 고민을 합니다. 바로 이 고민을 풀어 주는 똑똑한 방법이 이 단원의 주인공, 가상 메모리예요.

컴퓨터에는 기억을 담는 장치가 두 종류 있어요. 하나는 보조 기억 장치 — 용량이 크고, 전원을 꺼도 내용이 남는 '책장' 같은 곳입니다. 프로그램과 데이터를 영구히 보관하죠. 다른 하나는 주기억 장치 — 용량은 작지만 컴퓨터가 지금 당장 일을 처리하는 '책상'이에요. 흔히 '메모리'라고 부르는 게 바로 이 주기억 장치입니다. 컴퓨터는 필요한 프로그램을 책장(보조)에서 책상(주기억)으로 복사해 와서 실행해요. 이렇게 주기억 장치에 올라와 실행 중인 프로그램을 특별히 프로세스라고 부릅니다.

그런데 문제가 생깁니다. 책상(주기억 장치)은 좁아요. 여러 프로그램을 한꺼번에 돌리려면 그 많은 프로그램과 데이터를 전부 책상에 올려야 하는데, 자리가 모자라죠. 여기서 컴퓨터는 기막힌 꾀를 냅니다. "어차피 큰 프로그램이라도 지금 당장 쓰는 부분은 일부분뿐이잖아?" 그래서 당장 필요한 부분만 책상에 올리고, 나머지는 책장에 둔 채 책장의 일부를 마치 책상(메모리)처럼 빌려 쓰는 거예요. 이렇게 보조 기억 장치의 일부를 메모리처럼 쓰는 관리 기법이 바로 가상 메모리랍니다.

이 가상 메모리를 실제로 구현하는 대표적인 방법이 페이징이에요. 프로그램을 똑같은 크기의 조각인 '페이지'로 잘라 두고, 필요한 페이지만 골라서 주기억 장치에 올리는 방식이죠. 이때 주기억 장치 안에서 페이지가 들어갈 자리 하나하나를 '페이지 프레임'이라고 불러요. 페이지(조각)를 페이지 프레임(자리)에 끼워 넣는다고 생각하면 쉽습니다.

한마디로: 작은 책상으로도 큰 공부를 해내는 비결 = 다 펼치지 말고, 지금 쓸 조각(페이지)만 책상의 빈자리(페이지 프레임)에 올린다. 이것이 가상 메모리이고, 그 대표 방법이 페이징입니다.

이 단원에서 배울 것

이 단원의 중심에는 가상 메모리가 있습니다. 작은 주기억 장치로 큰 프로그램을 돌리는 메모리 관리 기법이죠. 세 세트는 이 큰 개념의 서로 다른 면을 비춥니다.

가상 메모리"작은 주기억 장치로 큰 프로그램을 돌리는 법"

세트 1

페이징이 일어나는 두 단계, 곧 프로그램을 페이지로 나누는 일과 그 페이지를 주기억 장치의 프레임에 올리는 일을 정점·자리·번호 같은 수치로 따라가 봅니다.

세트 2

보조 기억 장치, 주기억 장치, 그리고 이들을 묶어 운영하는 가상 메모리 시스템을 '구성 요소'의 관점에서 들여다봅니다. 각 장치가 무슨 일을 맡는지 정확히 가려내는 연습이에요.

세트 3

필요한 페이지를 프레임에 올리고, 자리가 없으면 한 페이지를 내보내는 과정을 차례차례 추적합니다. 이 과정의 허점과 그것을 막는 방법까지 함께 살핍니다.

세 세트를 차례로 풀고 나면, '왜 작은 메모리로도 큰 프로그램이 돌아가는가'라는 물음에 스스로 답할 수 있게 될 거예요.

❖ ❖ ❖
세트 1

컴퓨터가 큰 프로그램을 작은 주기억 장치로 실행하기 위해 쓰는 메모리 관리 기법을 가상 메모리라 한다. 가상 메모리를 실현하는 대표적인 방법인 페이징은 프로그램을 같은 크기의 작동 단위인 페이지로 분할한 뒤, 당장 실행에 필요한 페이지만을 주기억 장치에 올려 사용한다. 주기억 장치에서 페이지를 올릴 자리를 페이지 프레임이라 하며, 페이지 프레임의 크기는 페이지의 크기와 같아서 한 페이지는 한 페이지 프레임에 꼭 맞게 들어간다. 페이징은 프로그램을 페이지로 나누는 분할 단계와, 그 페이지를 프레임에 올리는 적재 단계로 이루어진다.

분할 단계에서는 프로그램 전체를 같은 크기의 페이지로 나눈다. 페이지의 크기는 한 프로그램 안에서 일정하며, 각 페이지에는 0부터 차례로 번호가 붙는다. 페이지의 크기를 모두 같게 하는 까닭은, 어떤 페이지든 어느 프레임에나 넣을 수 있게 하여 자리 배정을 단순하게 하기 위해서이다. 만약 페이지마다 크기가 다르다면 특정 페이지가 특정 프레임에만 들어갈 수 있어 빈자리를 효율적으로 쓰기 어렵다. 이 페이지 번호는 프로그램이 주기억 장치의 어느 자리에 올라가더라도 변하지 않는다. 페이지를 나눌 때 정해지는 페이지의 개수와 각 페이지의 크기는 프로그램 고유의 값으로, 프로그램이 실행되는 동안 달라지지 않는다. 다만 그 페이지들 가운데 어느 것이 주기억 장치에 올라가 있는지는 실행 도중 수시로 바뀐다.

적재 단계에서는 분할된 페이지 가운데 당장 필요한 것만을 골라 주기억 장치의 페이지 프레임에 올린다. 페이지 프레임도 0부터 번호가 매겨져 있으며, 그 개수는 주기억 장치의 크기에 따라 정해진다. 한 프로그램의 페이지들은 번호가 연속하더라도 반드시 연속한 페이지 프레임에 올라가는 것은 아니어서, 비어 있는 프레임이면 어디든 올라갈 수 있다. 예컨대 번호가 이어진 두 페이지가 서로 멀리 떨어진 프레임에 각각 올라갈 수도 있다. 이처럼 페이지의 번호와 그것이 놓인 프레임의 번호가 일치하지 않으므로, 어떤 페이지가 어느 프레임에 올라가 있는지를 알려면 페이지 번호와 페이지 프레임 번호의 대응을 따로 ㉠기록해 두는 표가 필요하다. 컴퓨터는 어떤 페이지를 찾을 때 이 표에서 그 페이지 번호에 짝지어진 프레임 번호를 읽어 실제 자리를 찾아간다. 페이지가 새로 올라가거나 내려갈 때마다 이 표의 내용도 그에 맞게 고쳐진다.

예를 들어 페이지의 크기가 100이고 주기억 장치에 페이지 프레임이 일곱 개 있다고 하자. 한 프로그램이 다섯 개의 페이지로 이루어져 있더라도, 그 가운데 당장 실행에 필요한 페이지가 두 개뿐이라면 두 개의 빈 프레임만 있으면 된다. 이때 0번 페이지가 0번 프레임에, 1번 페이지가 2번 프레임에 올라갔다면, 대응표에는 0번 페이지가 0번 프레임에, 1번 페이지가 2번 프레임에 대응한다는 정보가 기록된다. 컴퓨터가 1번 페이지를 찾을 때에는 이 표를 읽어 2번 프레임으로 찾아간다. 나머지 세 페이지는 프레임에 올라가지 않은 채로 있다가 필요해질 때 비로소 빈 프레임으로 불러오면 된다.

실행 중인 프로그램인 ㉡프로세스가 여럿일 때에도 페이징의 원리는 같다. 각 프로세스는 자신의 페이지들을 따로 분할해 가지며, 주기억 장치의 빈 페이지 프레임을 나누어 쓴다. 이때 한 페이지 프레임에는 한 번에 하나의 페이지만 올라가므로, 서로 다른 프로세스의 페이지가 같은 프레임을 동시에 차지하는 일은 없다. 또한 각 프로세스는 자신만의 대응표를 가지므로, 같은 번호의 페이지라도 어느 프로세스의 페이지인지에 따라 서로 다른 프레임에 놓인다. 페이지의 개수가 페이지 프레임의 개수보다 많아도, 당장 필요한 페이지의 수가 빈 프레임의 수를 넘지 않는 한 프로그램은 정상적으로 실행된다. 따라서 작은 주기억 장치만으로도 그보다 훨씬 많은 페이지로 이루어진 여러 프로그램을 함께 실행할 수 있다.

  • ① 가상 메모리는 큰 프로그램을 작은 주기억 장치로 실행하기 위한 메모리 관리 기법이다.
  • ② 한 프로그램에서 페이지의 크기는 일정하며 각 페이지에는 0부터 번호가 붙는다.

  • ④ 페이지 프레임의 개수는 주기억 장치의 크기에 따라 정해진다.
  • ⑤ 한 페이지 프레임에는 한 번에 하나의 페이지만 올라간다.

㉡에 대한 추론으로 적절한 것은?

  • ① 같은 프로세스의 페이지 번호가 연속하면, 그 페이지들이 올라가는 페이지 프레임의 번호도 반드시 연속한다.
  • ② 한 프로세스의 페이지 개수가 빈 페이지 프레임의 개수보다 많으면, 그 프로세스는 어떤 경우에도 실행될 수 없다.
  • ③ 두 프로세스가 동시에 실행될 때, 한 페이지 프레임에 두 프로세스의 페이지가 함께 올라가 자리를 나누어 쓴다.
  • ④ 한 프로세스가 가진 페이지 가운데 당장 필요한 페이지의 수가 빈 페이지 프레임의 수 이하이면, 그 페이지들은 주기억 장치에 올라갈 수 있다.
  • ⑤ 프로세스가 실행되는 동안 그 프로세스의 페이지 개수는 필요에 따라 늘어나거나 줄어든다.

정답 ④

3번 — 정답 ④ 프로세스(㉡)에 대한 추론입니다. 4문단 끝에 "당장 필요한 페이지의 수가 빈 프레임의 수를 넘지 않는 한 프로그램은 정상적으로 실행된다"라고 했죠. 이를 뒤집어 말한 ④가 정답입니다. ①은 "반드시 연속한 페이지 프레임에 올라가는 것은 아니"라는 3문단과 충돌하고, ②는 '필요한 페이지 수'가 아니라 '전체 페이지 수'로 잘못 따졌어요(전체가 많아도 실행될 수 있죠). ③은 "같은 프레임을 동시에 차지하는 일은 없다"와 반대, ⑤는 "페이지의 개수는 달라지지 않는다"(2문단)와 어긋납니다.

4번 [3점] — 정답 ⑤ 지문은 페이지가 "비어 있는 프레임이면 어디든 올라갈 수 있다"라고 했지, '페이지 번호와 같은 번호의 프레임에 올라가야 한다'고 한 적이 없어요. 〈보기〉에서도 을의 0번 페이지가 3번 프레임에 올라갔으니, 번호 일치 규칙은 성립하지 않습니다. 그래서 ⑤가 적절하지 않아 정답이에요. ①②는 '빈 프레임 자유 적재'·'비연속 적재'를, ③은 대응표의 역할을, ④는 7개 프레임 중 5개가 차서 1·6번 두 자리가 남는다는 계산을 정확히 적용한 적절한 이해입니다.

최근의 3D 애니메이션은 섬세한 입체 영상을 구현하여 실물을 촬영한 것 같은 느낌을 준다. 실물을 촬영하여 얻은 자연 영상을 그대로 화면에 표시할 때와 달리 3D 합성 영상을 생성, 출력하기 위해서는 모델링과 렌더링을 거쳐야 한다.

[모델링]은 3차원 가상 공간에서 물체의 모양과 크기, 공간적인 위치, 표면 특성 등과 관련된 고유의 값을 설정하거나 수정하는 단계이다. 모양과 크기를 설정할 때 주로 3개의 정점으로 형성되는 삼각형을 활용한다. 작은 삼각형의 조합으로 이루어진 그물과 같은 형태로 물체 표면을 표현하는 방식이다. 이 방법으로 복잡한 굴곡이 있는 표면도 정밀하게 표현할 수 있다. 이때 삼각형의 꼭짓점들은 물체의 모양과 크기를 결정하는 정점이 되는데, 이 정점들의 개수는 물체가 변형되어도 변하지 않으며, 정점들의 상대적 위치는 물체 고유의 모양이 변하지 않는 한 달라지지 않는다. 물체가 커지거나 작아지는 경우에는 정점 사이의 간격이 넓어지거나 좁아지고, 물체가 회전하거나 이동하는 경우에는 정점들이 간격을 유지하면서 회전축을 중심으로 회전하거나 동일 방향으로 동일 거리만큼 이동한다. 물체 표면을 구성하는 각 삼각형 면에는 고유의 색과 질감 등을 나타내는 표면 특성이 하나씩 지정된다.

공간에서의 입체에 대한 정보인 이 데이터를 활용하여, 물체를 어디에서 바라보는가를 나타내는 관찰 시점을 기준으로 2차원의 화면을 생성하는 것이 렌더링이다. 전체 화면을 잘게 나눈 점이 화소인데, 정해진 개수의 화소로 화면을 표시하고 각 화소별로 밝기나 색상 등을 나타내는 화솟값이 부여된다. 렌더링 단계에서는 화면 안에서 동일 물체라도 멀리 있는 경우는 작게, 가까이 있는 경우는 크게 보이는 원리를 활용하여 화솟값을 지정함으로써 물체의 원근감을 구현한다. 표면 특성을 나타내는 값을 바탕으로, 다른 물체에 가려짐이나 조명에 의해 물체 표면에 생기는 명암, 그림자 등을 고려하여 화솟값을 정해 줌으로써 물체의 입체감을 구현한다. 화면을 구성하는 모든 화소의 화솟값이 결정되면 하나의 프레임이 생성된다. 이를 화면출력장치를 통해 모니터에 표시하면 정지 영상이 완성된다.

모델링과 렌더링을 반복하여 생성된 프레임들을 순서대로 표시하면 동영상이 된다. 프레임을 생성할 때, 모델링과 관련된 계산을 완료한 후 그 결과를 이용하여 렌더링을 위한 계산을 한다. 이때 정점의 개수가 많을수록, 해상도가 높아 출력 화소의 수가 많을수록 연산 양이 많아져 연산 시간이 길어진다. 컴퓨터의 중앙처리장치(CPU)는 데이터 연산을 하나씩 순서대로 수행하기 때문에 과도한 양의 데이터가 집중되면 미처 연산되지 못한 데이터가 차례를 기다리는 병목 현상이 생겨 프레임이 완성되는 데 오랜 시간이 걸린다. CPU의 그래픽 처리 능력을 보완하기 위해 개발된 ㉠그래픽처리장치(GPU)는 연산을 비롯한 데이터 처리를 독립적으로 수행할 수 있는 장치인 코어를 수백에서 수천 개씩 탑재하고 있다. GPU의 각 코어는 그래픽 연산에 특화된 연산만을 할 수 있고 CPU의 코어에 비해서 저속으로 연산한다. 하지만 GPU는 동일한 연산을 여러 번 수행해야 하는 경우, 고속으로 출력 영상을 생성할 수 있다. 왜냐하면 GPU는 한 번의 연산에 쓰이는 데이터들을 순차적으로 각 코어에 전송한 후, 전체 코어에 하나의 연산 명령어를 전달하면, 각 코어는 모든 데이터를 동시에 연산하여 연산 시간이 짧아지기 때문이다.

컴퓨터에는 두 가지 기억 장치가 있다. 하나는 상대적으로 큰 저장 용량을 갖는 보조 기억 장치로, 프로그램이나 데이터를 영구적으로 저장하는 장치이다. 보조 기억 장치는 전원이 꺼져도 저장된 내용이 사라지지 않으나, 그 내용을 읽고 쓰는 속도는 느린 편이다. 다른 하나는 상대적으로 작은 저장 용량을 갖는 주기억 장치로, 컴퓨터가 당장 처리할 프로그램이나 데이터가 일시적으로 머무는 공간이며 흔히 ‘메모리’라고 불린다. 주기억 장치는 읽고 쓰는 속도가 빠르지만 용량이 작고, 전원이 꺼지면 머물러 있던 내용이 사라진다. 컴퓨터는 필요한 프로그램이나 데이터를 보조 기억 장치에서 복사하여 주기억 장치에 올린 뒤 처리하고, 처리가 끝나면 그것을 삭제하거나 다시 보조 기억 장치에 저장한다. 이처럼 두 장치는 용량과 속도, 보관의 지속성에서 서로 다른 특성을 지녀 각기 다른 역할을 맡는다.

프로그램의 연산을 ⓐ수행하는 중앙 처리 장치가 특정한 작업을 실행하려 할 때, 컴퓨터는 필요한 프로그램을 주기억 장치에 올린다. 이렇게 주기억 장치에 올라와 실행 중인 프로그램을 프로세스라 한다. 중앙 처리 장치는 보조 기억 장치의 내용을 곧바로 처리하지 못하고, 반드시 주기억 장치에 올라온 프로그램만을 처리한다. 그런데 여러 작업을 동시에 실행하려면 필요한 프로그램과 데이터를 모두 주기억 장치에 두어야 하지만, 주기억 장치는 용량이 작아 그러지 못할 수 있다. 프로그램 하나의 크기가 주기억 장치보다 큰 경우에는 그 프로그램조차 통째로 올릴 수 없다.

이 문제를 해결하기 위해 컴퓨터는 프로그램 중 당장 구동할 일부만을 주기억 장치에 올리고, 나머지는 보조 기억 장치에 둔 채 보조 기억 장치의 일부를 ‘메모리’처럼 ⓑ사용한다. 이러한 메모리 관리 기법을 가상 메모리라 한다. 가상 메모리는 별도의 새 장치가 아니라, 보조 기억 장치와 주기억 장치를 한데 묶어 운영하는 방식이다. 가상 메모리는 프로그램이 아무리 큰 용량을 ⓒ요구해도 중앙 처리 장치가 당장 실행하는 데 필요한 부분은 일부분이라는 점을 ⓓ이용한다. 프로그램 전체를 한꺼번에 처리하는 것이 아니라 부분마다 차례로 처리하므로, 그 순간순간 필요한 부분만 주기억 장치에 있으면 실행에 지장이 없는 것이다.

[A]가상 메모리는 페이징이라는 방법으로 실현된다. 페이징은 프로그램을 같은 크기의 작동 단위인 페이지로 분할하고, 페이지 단위로 프로그램을 주기억 장치에 올리는 방법이다. 주기억 장치에서 페이지를 올릴 자리를 페이지 프레임이라 한다. 큰 프로그램을 통째로 올리는 대신 이렇게 잘게 나누어 두면, 그중 당장 쓰이는 페이지만 골라 올릴 수 있다. 당장 필요하지 않은 페이지는 보조 기억 장치에 그대로 두고, 필요한 페이지만 주기억 장치의 빈 페이지 프레임으로 불러서 사용한다. 한 페이지 프레임에는 한 번에 한 페이지만 올라가며, 어떤 페이지가 어느 프레임에 올라가 있는지는 별도의 대응 정보로 관리된다. 실행 도중 다른 페이지가 필요해지면 그 페이지를 다시 불러오고, 자리가 모자라면 쓰지 않는 페이지를 보조 기억 장치로 되돌려 자리를 마련한다. 이로써 주기억 장치의 용량보다 큰 프로그램도 필요한 부분만 ⓔ나누어 올려 가며 실행할 수 있다.

이처럼 가상 메모리는 보조 기억 장치의 큰 용량과 주기억 장치의 빠른 처리를 함께 활용한다. 용량이 큰 보조 기억 장치는 프로그램 전체를 보관하는 역할을, 용량이 작은 주기억 장치는 당장 쓰이는 페이지를 담아 처리하는 역할을 맡는다. 사용자는 주기억 장치의 실제 용량이 얼마인지 의식하지 않고도, 마치 큰 메모리가 있는 것처럼 큰 프로그램을 실행할 수 있다. 이때 어느 페이지가 주기억 장치에 있고 어느 페이지가 보조 기억 장치에 있는지를 가려 필요한 페이지를 제때 옮기는 일은 모두 컴퓨터가 자동으로 처리하므로, 사용자가 직접 페이지를 옮길 필요는 없다. 두 장치가 페이징을 통해 맞물려 작동함으로써, 작은 주기억 장치만으로도 큰 프로그램의 실행이 가능해진다.

〈보기〉 어느 학생이 동영상 편집 프로그램과 문서 작성 프로그램을 한 컴퓨터에서 동시에 실행하였다. 두 프로그램의 전체 용량을 더하면 주기억 장치의 용량보다 훨씬 크지만, 두 프로그램은 멈추지 않고 함께 잘 작동하였다. 학생이 동영상을 편집하는 동안에는 그 작업에 필요한 페이지들이 주기억 장치에 올라가 있었고, 잠시 문서를 손보는 동안에는 문서 작성에 필요한 페이지들이 주기억 장치에 올라가 있었다.

  • ① 두 프로그램의 전체 용량이 주기억 장치보다 큰데도 함께 실행되는 것은, 당장 필요한 부분만 주기억 장치에 올리기 때문이겠군.
  • ② 동영상 편집에 쓰이지 않는 두 프로그램의 나머지 부분은 보조 기억 장치에 남아 있겠군.
  • ③ 동영상 편집 프로그램과 문서 작성 프로그램의 페이지들은 각각의 페이지 프레임에 올라가 처리되겠군.
  • ④ 작업의 초점이 동영상에서 문서로 옮겨 가면, 주기억 장치에 올라가 있는 페이지의 구성도 그에 맞게 바뀌겠군.
  • ⑤ 두 프로그램이 함께 작동하려면 두 프로그램의 모든 페이지가 동시에 주기억 장치에 올라가 있어야 하겠군.

가상 메모리는 프로그램을 같은 크기의 페이지로 나누어, 당장 필요한 페이지만 주기억 장치의 페이지 프레임에 올려 사용하는 메모리 관리 기법이다. 어떤 페이지가 어느 프레임에 올라가 있는지는 페이지 번호와 프레임 번호의 대응을 적어 둔 대응표로 관리된다. 대응표에는 각 페이지가 주기억 장치에 올라와 있는지 여부도 함께 적혀 있어, 컴퓨터는 이 표를 보고 필요한 페이지가 지금 주기억 장치에 있는지를 가려낸다. 그런데 모든 페이지가 주기억 장치에 올라가 있는 것은 아니므로, 중앙 처리 장치가 필요로 하는 페이지가 주기억 장치에 ㉠없는 경우가 생긴다. 이렇게 필요한 페이지가 주기억 장치에 없는 상태를 페이지 폴트라 한다. 페이지 폴트가 일어나면 그 페이지에 대한 처리는 곧바로 이어지지 못하고, 컴퓨터는 먼저 그 페이지를 보조 기억 장치에서 ㉡찾아내어 주기억 장치로 불러온 뒤에야 처리를 계속한다. 즉 페이지 폴트는 처리를 멈추게 하는 오류가 아니라, 필요한 페이지를 새로 불러오게 하는 신호인 셈이다.

페이지를 ㉢불러오는 정상적인 과정을 살펴보자. 중앙 처리 장치가 어떤 페이지를 요구하면, 컴퓨터는 먼저 그 페이지가 주기억 장치에 올라와 있는지를 대응표에서 확인한다. 올라와 있으면 페이지 폴트 없이 곧바로 그 자리를 ㉣찾아가 처리한다. 올라와 있지 않으면 페이지 폴트가 일어나며, 컴퓨터는 주기억 장치에 빈 페이지 프레임이 있는지 확인한다. 빈 프레임이 있으면 필요한 페이지를 보조 기억 장치에서 그 프레임으로 불러와 올린 뒤, 대응표에 그 페이지와 프레임의 대응을 새로 적어 갱신하고 처리한다. 이렇게 대응표를 갱신해 두어야 같은 페이지를 다음에 다시 요구할 때 곧바로 그 자리를 찾아갈 수 있다. 처음 요구된 페이지는 이처럼 보조 기억 장치에서 불러오는 과정을 한 번 거치지만, 그 뒤 주기억 장치에 머무는 동안에는 다시 불러오는 일 없이 빠르게 처리된다.

빈 페이지 프레임이 하나도 없을 때에는 한 단계가 더 ㉤더해진다. 이미 올라가 있는 페이지 가운데 하나를 골라 보조 기억 장치로 내보내 자리를 비운 다음, 그 자리에 필요한 페이지를 불러오는 것이다. 이렇게 자리를 비우기 위해 한 페이지를 내보내고 다른 페이지를 들이는 일을 페이지 교체라 한다. 페이지를 내보낼 때에는 그 페이지에 대응하던 기록을 대응표에서 지우고, 새로 들어온 페이지에 대한 기록을 더한다. 이때 어느 페이지를 내보낼지를 정하는 기준을 교체 기준이라 하는데, 한동안 쓰이지 않은 페이지를 먼저 내보내는 기준이 흔히 쓰인다. 한동안 쓰이지 않은 페이지는 앞으로도 한동안 쓰이지 않을 가능성이 높다고 보기 때문이다. 반대로 방금 쓰인 페이지는 곧 다시 쓰일 가능성이 높으므로 되도록 남겨 둔다. 내보낸 페이지는 사라지는 것이 아니라 보조 기억 장치에 그대로 남아, 나중에 다시 필요해지면 같은 과정을 거쳐 주기억 장치로 돌아온다.

페이지 교체는 작은 주기억 장치로 큰 프로그램을 돌리게 해 주지만, 교체가 잦아지면 그만큼 보조 기억 장치와 주기억 장치 사이에 페이지를 주고받는 일이 늘어 처리가 느려진다. 보조 기억 장치는 주기억 장치보다 읽고 쓰는 속도가 느려, 페이지를 한 번 주고받을 때마다 적지 않은 시간이 걸리기 때문이다. 특히 곧 다시 쓰일 페이지를 성급히 내보내면, 그 페이지를 금세 다시 불러와야 하므로 같은 페이지가 들고 나기를 거듭하게 된다. 이런 일이 반복되면 정작 연산에 쓸 시간보다 페이지를 옮기는 데 드는 시간이 더 많아져, 가상 메모리의 이점이 줄어든다. 빈 프레임의 수가 충분하면 교체 자체가 드물게 일어나므로 이런 지연도 덜하지만, 주기억 장치의 용량은 한정되어 있어 빈 프레임을 무한정 늘리는 데에는 한계가 있다. 그러므로 한정된 프레임을 어떻게 쓰느냐가 처리 속도를 좌우하며, 교체 기준을 잘 정하여 곧 쓰일 페이지는 되도록 주기억 장치에 남겨 두는 것이 중요하다.

DNS(도메인 네임 시스템) 스푸핑은 인터넷 사용자가 어떤 사이트에 접속하려 할 때 사용자를 위조 사이트로 접속시키는 행위를 말한다. 이는 도메인 네임을 IP 주소로 변환해 주는 과정에서 이루어진다.

인터넷에 연결된 컴퓨터들이 서로를 식별하고 통신하기 위해서 각 컴퓨터들은 IP(인터넷 프로토콜)에 따라 ㉠만들어지는 고유 IP 주소를 가져야 한다. 프로토콜은 컴퓨터들이 연결되어 서로 데이터를 주고받기 위해 사용하는 통신 규약으로 소프트웨어나 하드웨어로 구현된다. 현재 주로 사용하는 IP 주소는 ‘***.126.63.1’처럼 점으로 구분된 4개의 필드에 숫자를 사용하여 ㉡나타낸다. 이 주소를 중복 지정하거나 임의로 지정해서는 안 되고 공인 IP 주소를 부여받아야 한다.

공인 IP 주소에는 동일한 번호를 지속적으로 사용하는 고정 IP 주소와 번호가 변경되기도 하는 유동 IP 주소가 있다. 유동 IP 주소는 DHCP라는 프로토콜에 의해 부여된다. DHCP는 IP 주소가 필요한 컴퓨터의 요청을 받아 주소를 할당해 주고, 컴퓨터가 IP 주소를 사용하지 않으면 주소를 반환받아 다른 컴퓨터가 그 주소를 사용할 수 있도록 해 준다. 한편, 인터넷에 직접 접속은 안 되고 내부 네트워크에서만 서로를 식별할 수 있는 사설 IP 주소도 있다.

인터넷은 공인 IP 주소를 기반으로 동작하지만 우리가 인터넷을 사용할 때는 IP 주소 대신 사용하기 쉽게 ‘www..’ 등과 같이 문자로 ㉢이루어진 도메인 네임을 이용한다. 따라서 도메인 네임을 IP 주소로 변환해 주는 DNS가 필요하며 DNS를 운영하는 장치를 네임서버라고 한다. 컴퓨터에는 네임서버의 IP 주소가 기록되어 있어야 하는데, 유동 IP 주소를 할당받는 컴퓨터에는 IP 주소를 받을 때 네임서버의 IP 주소가 자동으로 기록되지만, 고정 IP 주소를 사용하는 컴퓨터에는 사용자가 네임서버의 IP 주소를 직접 기록해 놓아야 한다. 인터넷 통신사는 가입자들이 공동으로 사용할 수 있는 네임서버를 운영하고 있다.

㉮사용자가 어떤 사이트에 정상적으로 접속하는 과정을 살펴보자. 웹 사이트에 접속하려고 하는 컴퓨터를 클라이언트라 한다. 사용자가 방문하고자 하는 사이트의 도메인 네임을 주소창에 직접 입력하거나 포털 사이트에서 그 사이트를 검색해 클릭하면 클라이언트는 기록되어 있는 네임서버에 도메인 네임에 해당하는 IP 주소를 물어보는 질의 패킷을 보낸다. 네임서버는 해당 IP 주소가 자신의 목록에 있으면 클라이언트에 이 IP 주소를 알려 주는 응답 패킷을 보낸다. 응답 패킷에는 어느 질의 패킷에 대한 응답인지가 적혀 있다. 만일 해당 IP 주소가 목록에 없으면 네임서버는 다른 네임서버의 IP 주소를 알려 주는 응답 패킷을 보내고, 클라이언트는 다시 그 네임서버에 질의 패킷을 보내는 단계로 돌아가 같은 과정을 반복한다. 클라이언트는 이렇게 ㉣알아낸 IP 주소로 사이트를 찾아간다. 네임서버와 클라이언트는 UDP라는 프로토콜에 ㉤맞추어 패킷을 주고받는다. UDP는 패킷의 빠른 전송 속도를 확보하기 위해 상대에게 패킷을 보내기만 할 뿐 도착 여부는 확인하지 않으며, 특정 질의 패킷에 대해 처음 도착한 응답 패킷을 신뢰하고 다음에 도착한 패킷은 확인하지 않고 버린다. DNS 스푸핑은 UDP의 이런 허점들을 이용한다.

㉯DNS 스푸핑이 이루어지는 과정을 알아보자. 악성 코드에 감염되어 DNS 스푸핑을 행하는 컴퓨터를 공격자라 한다. 클라이언트가 네임서버에 특정 IP 주소를 묻는 질의 패킷을 보낼 때, 공격자에도 패킷이 전달되고 공격자는 위조 사이트의 IP 주소가 적힌 응답 패킷을 클라이언트에 보낸다. 공격자가 보낸 응답 패킷이 네임서버가 보낸 응답 패킷보다 클라이언트에 먼저 도착하고 클라이언트는 공격자가 보낸 응답 패킷을 옳은 패킷으로 인식하여 위조 사이트로 연결된다.
